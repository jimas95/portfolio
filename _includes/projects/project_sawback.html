Sawback Pick and Place

ROS | SLAM | Autonomous Navigation | C++ | MoveIt | Robitc Manipulation

Description

This project is a proof of concept that we are able withought any human intervention navigate in a room, detect objects that are not where they should be and return them to their original position. The robot is able to navigate around the lab space autonomous, there is a mode of patrol, when this mode is activated the robot will go back and forth in a certain area. If an item is detected then the robot will pick the object and based on which object it detected it would autonomous navigate to the area of drawers,box container or shelfs and place the object where it belongs.

Demonstration of sawback opening a drawer

Overview

The hardest part on this project was the intergration of all the compenents(robots and sensors) we have and fuse them together. The robot named sawback is actually 2 different robots. The first one is Ridgeback an omnidirectional mobile platform on which the robot Sawyer was attached to it, thus the name of our robot is Sawback. Each robot has his own computer and if we add the developers computer we have in total 3 different computers that each one has to execute specific instructions and comunicate with the others. Another problem to overcome was packages that were used, in total there where almost 80 different packages! Split in different workspaces. So extented workspaces had to be created that link the whole project together and machine tags in order to specifiy what is being executed where.

Autonomous Navigation

As a first step in this project I created the map of the room that I was working on(Figure 1). This room was the lab of my Univerisity. In order to create the map, Velodyne lidar sensor was used and library rtabmaps in order to SLAM and convert our 3D point cloud to a 2D map.

Fig. 1 Map Creation

After the creation of the map (Figure 2) specific points where saved. MoveBase action was used, Hokoyu front/rear and Velodyne lidar sensors help us to avoid any colisionts. The points that where saved are :

  • Kitchen Area
  • Charging Station
  • Shelfs Area
  • Box container Area
  • Drawer Area
At these point we are able to navigate around(with no collision) the lab and go on each of these location by just calling a service and the location we want.

Fig. 2 Lab Map

Patrol

In the project there is a special mode that gets activated with a service. When patrol mode (Figure 3) is on the robot will start from its original position and go forward for about 5 meters and the return to the starting position again and again. If an object is detected then the robot will stop and the robotic arm manipulation will start.

Fig 3 Patrol MODE.

Robotic Arm Manipulation

In order to pick the objects I wanted to use some sorth of color detection and the gpd library (grasping position depth) but due to limited time I placed an April Tag on each object. The robot when the april tag gets detected will get allinged with it an stop in a set distance from it. Moveit cpp was used in order to manipulate the arm and grasp the object. At Figure 4 you can see some examples of grabing a dice cube.

Fig 4 Pick Item.

When the object has being picked from the arm, then based on which object we got we will navigate to the proper location and then place it in a specific container. This container could be a box on top of a table, shelfs or a drawer. More specific for the drawer manipulation because it requires some extra precision to open it, an extra April Tag was used in order to know the exact position of the drawer. For the detection of AprilTag a bumblebee camera was used facing the floor.
